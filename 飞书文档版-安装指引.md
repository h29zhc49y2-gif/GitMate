````markdown
# GitMate

## 任务目标

读取 Git 暂存区 / 分支差异，自动生成符合 Conventional Commits 规范的提交信息与 Pull Request 描述。

## 执行流程

### Step 1：读取变更

```bash
python scripts/commit_generator.py --detect
```

### Step 2：推断提交类型

| 变更特征 | 推断 type |
|---------|----------|
| 新增文件为主 | `feat` |
| 修改现有文件 | `fix` |
| 仅 .md 文件 | `docs` |
| 配置文件 | `chore` |
| 测试文件 | `test` |
| 重构 | `refactor` |
| CSS/样式 | `style` |
| 性能关键词 | `perf` |

### Step 3：生成提交信息

```bash
python scripts/commit_generator.py --lang zh
```

### Step 4：生成 PR 描述

```bash
python scripts/pr_generator.py --base main --lang zh
```

---

## 核心规则

### 规则 1：Conventional Commits 格式

```
<type>(<scope>): <subject>

<body>

<footer>
```

### 规则 2-6

- 自动从文件路径推断 scope
- subject 动词开头、50 字符内
- body 逐文件列变更条目
- 兼容 commitlint 规范
- 中文/英文双语言支持

## 输出示例

```
feat(home): 添加用户列表组件

- 新增 UserList.tsx
- 新增 UserCard.tsx
- 修改 index.ts
```
````
